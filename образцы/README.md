# Образцы протоколов заседаний

Шаблоны структурированы по типовой форме делопроизводства РК
(СТ РК 1042-2001 / типовая структура: «СЛУШАЛИ — ВЫСТУПИЛИ — ПОСТАНОВИЛИ»).

Каждый шаблон — отдельный `.md` файл с:

1. **YAML-фронтматтером** — машиночитаемые метаданные (`id`, `name`, `description`, `language`, `sections`) для подгрузки бэкендом и UI-селектора.
2. **Body c плейсхолдерами `{{...}}`** — каркас, который заполняется LLM из транскрипта.

## Доступные шаблоны

| ID                       | Название                                       | Когда использовать                                  |
|--------------------------|------------------------------------------------|-----------------------------------------------------|
| `general`                | Общий протокол заседания                       | Универсальный, если нет специальной формы           |
| `short`                  | Краткий протокол                               | Оперативные совещания, планёрки                     |
| `attestation`            | Заседание аттестационной комиссии              | Аттестация работников                               |
| `disciplinary`           | Заседание дисциплинарной комиссии              | Рассмотрение дисциплинарных проступков              |
| `accident-investigation` | Расследование несчастного случая на производстве | Комиссия по расследованию НС (форма Н-1)         |
| `general-assembly`       | Общее собрание участников (учредителей)        | ТОО / АО / ОСИ                                      |
| `tender`                 | Заседание тендерной (конкурсной) комиссии      | Госзакупки, выбор поставщика                        |
| `pedagogical-council`    | Заседание педагогического совета               | Школа / колледж / вуз                               |

## Источник списка для UI

`manifest.json` — авторитетный реестр шаблонов (id, file, name, description).
UI и бэкенд должны читать его, а не хардкодить список.

## Как добавить новый шаблон

1. Создать `<id>.md` со фронтматтером (поля: `id`, `name`, `description`, `language`, `sections`).
2. Добавить запись в `manifest.json`.
3. Тело шаблона использовать только заранее объявленные плейсхолдеры — список см. в `placeholders.md`.
