# Плейсхолдеры

Список зарезервированных переменных, которые LLM подставляет в шаблон при
генерации протокола. Бэкенд должен передавать эти ключи в JSON-схеме
структурированного вывода (`responses.parse`).

## Шапка
- `{{org}}` — наименование организации
- `{{body}}` — наименование органа (комиссии, совета, собрания)
- `{{number}}` — номер протокола
- `{{city}}` — город / нас. пункт
- `{{date}}` — дата заседания (DD.MM.YYYY)
- `{{time_start}}` — время начала
- `{{time_end}}` — время окончания
- `{{place}}` — место проведения

## Лица
- `{{chair}}` — председатель
- `{{secretary}}` — секретарь
- `{{present}}` — список присутствующих (строки «ФИО — должность»)
- `{{absent}}` — список отсутствующих
- `{{invited}}` — приглашённые
- `{{quorum}}` — отметка о кворуме («Кворум имеется» / «Кворум отсутствует»)

## Повестка и вопросы
- `{{agenda}}` — упорядоченный список вопросов повестки дня
- `{{items[]}}` — массив рассмотренных вопросов; для каждого:
  - `item.no` — номер вопроса
  - `item.title` — формулировка вопроса
  - `item.heard` — кто и о чём докладывал (СЛУШАЛИ)
  - `item.spoke` — выступления (ВЫСТУПИЛИ) — массив `{speaker, text}`
  - `item.vote` — `{for, against, abstain}`
  - `item.resolved` — формулировка постановления (ПОСТАНОВИЛИ)

## Спец-блоки
- `{{action_items[]}}` — поручения: `{task, assignee, deadline}`
- `{{accident}}` — для шаблона расследования НС: `{victim, date, place, circumstances, causes, conclusion}`
- `{{candidates[]}}` — для аттестации/тендера: `{name, details, decision}`

## Подписи
- `{{signatures}}` — подписи председателя и секретаря (формальный блок в конце)

## Правила заполнения
- Если данных нет — оставлять поле **пустым**, не выдумывать.
- Списки — по фактическим упоминаниям в транскрипте.
- Голосование — только если явно прозвучало («за — 5, против — 0, воздержался — 1»).
